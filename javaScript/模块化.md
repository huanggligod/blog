## 全局function
    + 问题：污染全局命名空间，容易引起命名冲突或数据不安全，而且模块成员之间看不出直接关系
```
    function m1() {
        //...
    }
    function m2() {
        //...
    }
```    

## 简单对象封装
    + 作用：减少了全局变量，解决命名冲突
    + 问题：数据不安全，这个写法会暴露内部状态，外部代码可以直接改写
```
    let myModule = {
        _count: 0,
        f00() {
            console.log('${this._count}')
        },
        bar() {
            console.log('bar')
        }
    }
    myModule._count = 1
    myModule.foo() // 1
```
